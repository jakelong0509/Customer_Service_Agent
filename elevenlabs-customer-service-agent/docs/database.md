# Database structure

Schema for the tool API: **customers**, **provider_names** (Doctor, Nurse, Room, Equipment), **providers** (bookable resources), **slot_templates** (fixed 30-minute grid with lunch omitted), **general_statuses** (pending, scheduled, completed, cancelled), **appointments**, **callback_requests**, **appointment_resource_bookings** (per-resource reservations to prevent double booking), plus **RAG/clinical** tables (**clinical_note_embeddings**, **umls_concepts**, **rxnorm_***, **drug_interactions**). Postgres, managed via asyncpg.

RXNCONSO lives in Milvus (Zilliz), not PostgreSQL — see `docs/RAG.md`.

---

## Overview

```
customers
    |
    +--< callback_requests --> general_statuses
    +--< appointments -----> general_statuses
              |
              +--< appointment_resource_bookings --> providers --> provider_names
                                          |
                                          +-- slot_templates
```

- **No pre-generated “available” rows.** `appointment_resource_bookings` rows are inserted when an appointment is **confirmed** (one row per reserved resource per 30-minute slot).
- **Multi-resource visits:** the same `appointment_id` appears on several booking rows (e.g. doctor + exam room). A **unique** constraint on `(provider_id, booking_date, slot_template_id)` stops the same resource from being booked twice for that slot.
- **Slot templates** use **clinic-local wall-clock** `TIME` values. `appointments.scheduled_at` is **`TIMESTAMPTZ`** (typically UTC in the app). The app must keep `booking_date` + `slot_template_id` consistent with `scheduled_at` (and your chosen clinic timezone). Demo seeds may assume UTC = local for simplicity.
- **Statuses** (`callback_requests`, `appointments`, `appointment_resource_bookings`) reference **`general_statuses`** by integer id (seeded: pending, scheduled, completed, cancelled).

## Tables

### `customers`

| Column       | Type         | Nullable | Description                |
|-------------|--------------|----------|----------------------------|
| id          | UUID         | no       | Primary key (default gen_random_uuid()) |
| phone       | VARCHAR(32)  | yes      | E.164 or normalized       |
| email       | VARCHAR(255) | yes      |                            |
| name        | VARCHAR(255) | yes      | Display name               |
| plan        | VARCHAR(64)  | yes      | e.g. standard, premium     |
| status      | VARCHAR(32)  | no       | Default `'active'`         |
| created_at  | TIMESTAMPTZ  | no       | Default now()              |
| updated_at  | TIMESTAMPTZ  | no       | Default now()              |

- Unique index on `phone` (where not null).
- Used by: `lookup_customer`, `get_account_info`, and as FK for callbacks and appointments.

---

### `provider_names`

| Column     | Type         | Nullable | Description                          |
|-----------|--------------|----------|--------------------------------------|
| id        | INTEGER      | no       | Primary key (GENERATED BY DEFAULT AS IDENTITY) |
| name      | VARCHAR(255) | no       | e.g. Doctor, Nurse, Room, Equipment |
| created_at| TIMESTAMPTZ  | no       | Default now()                        |

- Unique index on `name`.
- Seeded rows: `Doctor`, `Nurse`, `Room`, `Equipment`.

---

### `providers`

| Column       | Type         | Nullable | Description                |
|-------------|--------------|----------|----------------------------|
| id          | UUID         | no       | Primary key                |
| kind        | INTEGER      | no       | FK → `provider_names.id`   |
| name        | VARCHAR(255) | no       | Display name               |
| active      | BOOLEAN      | no       | Default true               |
| created_at  | TIMESTAMPTZ  | no       | Default now()              |

- Index on `kind` where `active` (for filtering bookable resources).

---

### `slot_templates`

| Column       | Type         | Nullable | Description                |
|-------------|--------------|----------|----------------------------|
| id          | SMALLINT     | no       | Primary key (seeded 1–12)  |
| start_time  | TIME         | no       | Start of 30-minute bucket; unique |

- Seeded in **`app/init_db/create_tables.sql`**: morning **09:00–12:00** and afternoon **14:00–17:00** in 30-minute steps (**13:00–14:00 lunch** has no template row).
- Availability search generates candidates from these times (plus rules for days off, etc.) and checks for **missing** booking rows per required provider.

---

### `general_statuses`

| Column     | Type         | Nullable | Description                          |
|-----------|--------------|----------|--------------------------------------|
| id        | INTEGER      | no       | Primary key (GENERATED BY DEFAULT AS IDENTITY) |
| name      | VARCHAR(255) | no       | Status label                         |
| created_at| TIMESTAMPTZ  | no       | Default now()                        |

- Unique index on `name`.
- Seeded: `pending`, `scheduled`, `completed`, `cancelled`.
- Referenced by `callback_requests.status`, `appointments.status`, `appointment_resource_bookings.status`.

---

### `callback_requests`

| Column        | Type         | Nullable | Description                |
|---------------|--------------|----------|----------------------------|
| id            | UUID         | no       | Primary key                |
| customer_id   | UUID         | yes      | FK → customers.id          |
| phone         | VARCHAR(32)  | no       | Number to call back        |
| requested_for | VARCHAR(128) | yes      | e.g. "next available", "tomorrow 2pm" |
| status        | INTEGER      | no       | FK → `general_statuses.id` |
| created_at    | TIMESTAMPTZ  | no       | Default now()              |

- Used by: `schedule_callback`.

---

### `appointments`

| Column       | Type         | Nullable | Description                |
|-------------|--------------|----------|----------------------------|
| id          | UUID         | no       | Primary key                |
| customer_id | UUID         | yes      | FK → customers.id          |
| scheduled_at| TIMESTAMPTZ  | no       | Start of the visit         |
| subject     | VARCHAR(255) | no       | e.g. visit reason / label  |
| status      | INTEGER      | no       | FK → `general_statuses.id` |
| notes       | TEXT         | yes      |                            |
| created_at  | TIMESTAMPTZ  | no       | Default now()              |

- Indexes on `scheduled_at`, `customer_id`.
- Used with **`appointment_resource_bookings`** when reserving doctors, rooms, or equipment.

---

### `appointment_resource_bookings`

| Column           | Type         | Nullable | Description                |
|------------------|--------------|----------|----------------------------|
| id               | UUID         | no       | Primary key                |
| appointment_id   | UUID         | no       | FK → appointments.id (CASCADE) |
| provider_id      | UUID         | no       | FK → providers.id (RESTRICT) |
| booking_date     | DATE         | no       | Calendar date for the slot |
| slot_template_id | SMALLINT     | no       | FK → slot_templates.id (RESTRICT) |
| status           | INTEGER      | no       | FK → `general_statuses.id` |
| created_at       | TIMESTAMPTZ  | no       | Default now()              |

- **Unique** `(provider_id, booking_date, slot_template_id)` — prevents double booking per resource.
- Indexes for availability lookups and by `appointment_id`.

## RAG / clinical tables (PostgreSQL)

Vectors for clinical notes and UMLS live in Milvus collections (e.g. `clinical_note_vectors`, `umls_vectors`); these tables hold relational metadata and RxNorm caches only.

### `clinical_note_embeddings`

| Column      | Type        | Nullable | Description                    |
|------------|-------------|----------|--------------------------------|
| id         | SERIAL      | no       | Primary key                    |
| note_hash  | VARCHAR(64) | yes      | Unique when set                |
| icd10_codes| TEXT[]      | yes      |                                |
| cpt_codes  | TEXT[]      | yes      |                                |
| created_at | TIMESTAMPTZ | no       | Default now()                  |

### `umls_concepts`

| Column              | Type         | Nullable | Description   |
|---------------------|--------------|----------|---------------|
| cui                 | VARCHAR(8)   | no       | Primary key   |
| concept_name        | TEXT         | yes      |               |
| semantic_type       | VARCHAR(10)  | yes      |               |
| source_vocabularies | TEXT[]       | yes      |               |

### `rxnorm_relationships` (RXNREL)

Brand↔generic, ingredient, dose form, etc. Columns include: `rxcui1`, `rxaui1`, `stype1`, `rel`, `rxcui2`, `rxaui2`, `stype2`, `rela`, `rui`, `srui`, `sab`, `sl`, `rg`, `dir`, `suppress`, `cvf`. Primary key: `id` (IDENTITY).

### `rxnorm_attributes` (RXNSAT)

Attributes, NDC codes, RxNorm-specific properties. Columns include: `rxcui`, `lui`, `sui`, `rxaui`, `stype`, `code`, `atui`, `satui`, `atn`, `sab`, `atv`, `suppress`, `cvf`. Primary key: `id` (IDENTITY).

### `rxnorm_semantic_types` (RXNSTY)

| Column | Type         | Nullable |
|--------|--------------|----------|
| id     | INTEGER      | no (IDENTITY) |
| rxcui  | VARCHAR(8)   | yes      |
| tui    | VARCHAR(10)  | yes      |
| stn    | VARCHAR(30)  | yes      |
| sty    | VARCHAR(100) | yes      |
| atui   | VARCHAR(12)  | yes      |
| cvf    | VARCHAR(10)  | yes      |

### `rxnorm_documentation` (RXNDOC)

Abbreviation / reference lookup: `key_val`, `value_val`, `type_val`, `expl`. Primary key: `id` (IDENTITY).

### `drug_interactions`

| Column        | Type         | Nullable |
|---------------|--------------|----------|
| id            | SERIAL       | no       |
| drug1_rxcui   | VARCHAR(8)   | yes      |
| drug2_rxcui   | VARCHAR(8)   | yes      |
| severity      | VARCHAR(20)  | yes      |
| description   | TEXT         | yes      |
| mechanism     | TEXT         | yes      |

### RxNorm indexes (PostgreSQL)

- `idx_rxnorm_relationships_rxcui1`, `idx_rxnorm_relationships_rxcui2`, `idx_rxnorm_relationships_rel`, `idx_rxnorm_relationships_rela`
- `idx_rxnorm_attributes_rxcui`, `idx_rxnorm_attributes_atn`
- `idx_rxnorm_semantic_types_rxcui`

## SQL (create tables)

Script file: **`app/init_db/create_tables.sql`** — run with `psql $DATABASE_URL -f app/init_db/create_tables.sql` or paste the block below.

```sql
-- Create tables for Customer Service Agent (see docs/database.md)
-- Run against your Postgres DB, e.g. psql $DATABASE_URL -f app/init_db/create_tables.sql

-- CREATE EXTENSION IF NOT EXISTS "pgcrypto";

CREATE TABLE customers (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    phone       VARCHAR(32),
    email       VARCHAR(255),
    name        VARCHAR(255),
    plan        VARCHAR(64),
    status      VARCHAR(32) NOT NULL DEFAULT 'active',
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX idx_customers_phone ON customers (phone) WHERE phone IS NOT NULL;

-- Bookable resources: clinicians, rooms, equipment (shared clinic calendar per resource).
CREATE TABLE provider_names (
    id         INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    name       VARCHAR(255) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX idx_provider_names_name ON provider_names (name);
INSERT INTO provider_names (name) VALUES ('Doctor'), ('Nurse'), ('Room'), ('Equipment');

CREATE TABLE providers (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    kind       INTEGER NOT NULL REFERENCES provider_names (id),
    name       VARCHAR(255) NOT NULL,
    active     BOOLEAN NOT NULL DEFAULT true,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_providers_kind ON providers (kind) WHERE active = true;


-- 30-minute slot starts (clinic-local wall clock). Lunch 13:00–14:00 omitted from templates.
CREATE TABLE slot_templates (
    id         SMALLINT PRIMARY KEY,
    start_time TIME NOT NULL UNIQUE
);

INSERT INTO slot_templates (id, start_time) VALUES
    (1, '09:00'), (2, '09:30'), (3, '10:00'), (4, '10:30'), (5, '11:00'), (6, '11:30'),
    (7, '14:00'), (8, '14:30'), (9, '15:00'), (10, '15:30'), (11, '16:00'), (12, '16:30')
ON CONFLICT (id) DO NOTHING;


CREATE TABLE general_statuses (
    id         INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    name       VARCHAR(255) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX idx_general_statuses_name ON general_statuses (name);
INSERT INTO general_statuses (name) VALUES ('pending'), ('scheduled'), ('completed'), ('cancelled');

CREATE TABLE callback_requests (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id   UUID REFERENCES customers (id) ON DELETE SET NULL,
    phone         VARCHAR(32) NOT NULL,
    requested_for VARCHAR(128),
    status        INTEGER NOT NULL REFERENCES general_statuses (id),
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE appointments (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id  UUID REFERENCES customers (id) ON DELETE SET NULL,
    scheduled_at TIMESTAMPTZ NOT NULL,
    subject      VARCHAR(255) NOT NULL,
    status       INTEGER NOT NULL REFERENCES general_statuses (id),
    notes        TEXT,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_appointments_scheduled_at ON appointments (scheduled_at);
CREATE INDEX idx_appointments_customer_id ON appointments (customer_id);

-- One row per resource reserved for an appointment (created on confirm). No pre-generated “available” rows.
-- UNIQUE (provider_id, booking_date, slot_template_id) prevents double-booking per resource per slot.
CREATE TABLE appointment_resource_bookings (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    appointment_id   UUID NOT NULL REFERENCES appointments (id) ON DELETE CASCADE,
    provider_id      UUID NOT NULL REFERENCES providers (id) ON DELETE RESTRICT,
    booking_date     DATE NOT NULL,
    slot_template_id SMALLINT NOT NULL REFERENCES slot_templates (id) ON DELETE RESTRICT,
    status           INTEGER NOT NULL REFERENCES general_statuses (id),
    created_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (provider_id, booking_date, slot_template_id)
);
CREATE INDEX idx_appointment_resource_bookings_lookup
    ON appointment_resource_bookings (provider_id, booking_date);
CREATE INDEX idx_appointment_resource_bookings_appointment
    ON appointment_resource_bookings (appointment_id);


-- ============================================================================
-- RAG / Clinical Tables
-- See docs/RAG.md for full architecture.
-- RXNCONSO is stored entirely in Milvus (Zilliz) — NOT in PostgreSQL.
-- The following tables are PostgreSQL-only (no vectors, pure relational).
-- ============================================================================

-- Clinical note metadata (vectors stored in Milvus: clinical_note_vectors)
CREATE TABLE clinical_note_embeddings (
    id          SERIAL PRIMARY KEY,
    note_hash   VARCHAR(64) UNIQUE,
    icd10_codes TEXT[],
    cpt_codes   TEXT[],
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- UMLS concept cache (vectors stored in Milvus: umls_vectors)
CREATE TABLE umls_concepts (
    cui                 VARCHAR(8) PRIMARY KEY,
    concept_name        TEXT,
    semantic_type       VARCHAR(10),
    source_vocabularies TEXT[]
);

-- RXNREL: concept relationships (brand↔generic, ingredient, dose form)
CREATE TABLE rxnorm_relationships (
    id        INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    rxcui1    VARCHAR(8),
    rxaui1    VARCHAR(8),
    stype1    VARCHAR(10),
    rel       VARCHAR(10),
    rxcui2    VARCHAR(8),
    rxaui2    VARCHAR(8),
    stype2    VARCHAR(10),
    rela      VARCHAR(50),
    rui       VARCHAR(12),
    srui      VARCHAR(12),
    sab       VARCHAR(20),
    sl        VARCHAR(50),
    rg        VARCHAR(12),
    dir       VARCHAR(1),
    suppress  VARCHAR(1),
    cvf       VARCHAR(10)
);

-- RXNSAT: attributes, NDC codes, RxNORM-specific properties
CREATE TABLE rxnorm_attributes (
    id       INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    rxcui    VARCHAR(8),
    lui      VARCHAR(8),
    sui      VARCHAR(8),
    rxaui    VARCHAR(8),
    stype    VARCHAR(10),
    code     VARCHAR(20),
    atui     VARCHAR(12),
    satui    VARCHAR(12),
    atn      VARCHAR(50),
    sab      VARCHAR(20),
    atv      TEXT,
    suppress VARCHAR(1),
    cvf      VARCHAR(10)
);

-- RXNSTY: semantic type classification
CREATE TABLE rxnorm_semantic_types (
    id    INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    rxcui VARCHAR(8),
    tui   VARCHAR(10),
    stn   VARCHAR(30),
    sty   VARCHAR(100),
    atui  VARCHAR(12),
    cvf   VARCHAR(10)
);

-- RXNDOC: abbreviation / reference lookup
CREATE TABLE rxnorm_documentation (
    id         INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    key_val   VARCHAR(100),
    value_val TEXT,
    type_val  VARCHAR(10),
    expl      TEXT
);

-- Drug interaction cache
CREATE TABLE drug_interactions (
    id           SERIAL PRIMARY KEY,
    drug1_rxcui  VARCHAR(8),
    drug2_rxcui  VARCHAR(8),
    severity     VARCHAR(20),
    description  TEXT,
    mechanism    TEXT
);

-- RxNorm relational indexes
CREATE INDEX idx_rxnorm_relationships_rxcui1 ON rxnorm_relationships (rxcui1);
CREATE INDEX idx_rxnorm_relationships_rxcui2 ON rxnorm_relationships (rxcui2);
CREATE INDEX idx_rxnorm_relationships_rel    ON rxnorm_relationships (rel);
CREATE INDEX idx_rxnorm_relationships_rela   ON rxnorm_relationships (rela);
CREATE INDEX idx_rxnorm_attributes_rxcui     ON rxnorm_attributes (rxcui);
CREATE INDEX idx_rxnorm_attributes_atn       ON rxnorm_attributes (atn);
CREATE INDEX idx_rxnorm_semantic_types_rxcui ON rxnorm_semantic_types (rxcui);
```

## Notes

- UUID PKs for customers, providers, appointments, bookings, callbacks; **SMALLINT** for `slot_templates.id`; **SERIAL** / **IDENTITY** for clinical and RxNorm helper tables as in the script.
- Timestamps are UTC (`TIMESTAMPTZ`).
- Optional FKs use `ON DELETE SET NULL` on `customers` so deleting a customer does not cascade to appointments unless you change that later.
- **Lunch, sick days, vacation:** model with application rules and/or a future `provider_unavailability` table; they are not stored as rows in `appointment_resource_bookings`.
- Add further indexes when you filter heavily on status columns or date ranges.
